# 服务器部署指南

本文档说明如何将 **脊柱侧弯康复训练** 项目的 **NestJS 后端 API** 部署到 Linux 云服务器（阿里云 / 腾讯云 / 华为云等），并与 **MySQL**、**MinIO**、**Nginx HTTPS** 及 **微信小程序** 联调上线。

> 小程序前端编译与上传见文末「小程序发布」；本地开发见 [本地安装-无Docker.md](本地安装-无Docker.md)；环境变量详解见 [环境配置说明.md](环境配置说明.md)。

---

## 一、架构概览

```
┌─────────────┐     HTTPS      ┌──────────────┐
│ 微信小程序   │ ─────────────► │ Nginx :443   │
└─────────────┘                │  api.xxx.com │
                               └──────┬───────┘
                                      │ 反代
                               ┌──────▼───────┐
                               │ NestJS :3000 │
                               │ /api/v1      │
                               └──────┬───────┘
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
              ┌──────────┐    ┌──────────┐    ┌──────────────────┐
              │ MySQL 8  │    │ MinIO    │    │ 微信开放平台 API  │
              │ spine_   │    │ spine-   │    │ (登录/订阅消息)   │
              │ train    │    │ train 桶 │    └──────────────────┘
              └──────────┘    └──────────┘
                    ▲
                    │ downloadFile 合法域名（视频直链）
              用户手机播放训练视频
```

| 组件 | 作用 | 生产建议 |
|------|------|----------|
| **NestJS API** | 业务接口，全局前缀 `api/v1` | PM2 守护，仅内网或本机监听 |
| **MySQL 8** | 用户、计划、训练记录等 | 独立实例或 RDS，专用账号 |
| **MinIO** | 训练视频、头像对象存储 | HTTPS 公网前缀，配置 downloadFile 域名 |
| **Nginx** | HTTPS 终结、反向代理 | Let's Encrypt 免费证书 |
| **定时任务** | 订阅消息提醒（每分钟 Cron） | 服务器时区设为 `Asia/Shanghai` |

## 二、服务器与软件要求

### 2.1 推荐配置

| 项目 | 最低 | 推荐（小规模） |
|------|------|----------------|
| CPU | 1 核 | 2 核 |
| 内存 | 2 GB | 4 GB |
| 系统盘 | 40 GB | 60 GB+（含 MinIO 视频） |
| 操作系统 | Ubuntu 22.04 LTS / Debian 12 / CentOS Stream 9 | Ubuntu 22.04 |

### 2.2 软件版本

| 软件 | 版本 |
|------|------|
| **Node.js** | ≥ 18.18（推荐 **20.x LTS**） |
| **npm** | ≥ 9 |
| **MySQL** | 8.0+ |
| **MinIO** | 最新稳定版（或 Docker 镜像） |
| **Nginx** | 1.18+ |
| **PM2** | 最新（进程守护，可选但推荐） |

### 2.3 域名与备案

- API 域名示例：`api.example.com` → 解析到云服务器公网 IP。
- 小程序 **request 合法域名** 必须为 **HTTPS**，且域名需 **ICP 备案**（国内服务器）。
- 若训练视频通过 MinIO 公网地址播放，还需在公众平台配置 **downloadFile 合法域名**（或使用 CDN / 同一主域名反代 MinIO）。

## 三、部署前准备

### 3.1 开放端口（安全组 / 防火墙）

| 端口 | 用途 | 是否对公网开放 |
|------|------|----------------|
| 22 | SSH | 是（建议改密钥登录、限制 IP） |
| 80 | HTTP（证书申请、跳转 HTTPS） | 是 |
| 443 | HTTPS（Nginx） | 是 |
| 3000 | NestJS | **否**（仅 Nginx 反代访问） |
| 3306 | MySQL | **否**（仅本机或内网） |
| 9000 / 9001 | MinIO API / 控制台 | 9000 按需（或通过 Nginx 反代）；控制台建议仅内网 |

```bash
# Ubuntu 示例（ufw）
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 3.2 安装 Node.js 20（Ubuntu 示例）

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # v20.x
npm -v
```

### 3.3 安装 Git 与拉取代码

```bash
sudo apt-get install -y git
cd /opt
sudo git clone <你的仓库地址> training-plan
sudo chown -R $USER:$USER training-plan
cd training-plan
```

也可使用 `scp` / `rsync` 上传 `server` 目录，但需包含 `prisma/migrations`。

## 四、MySQL 部署

### 4.1 安装 MySQL 8（Ubuntu）

```bash
sudo apt-get install -y mysql-server
sudo mysql_secure_installation
```

### 4.2 建库与专用账号

```sql
CREATE DATABASE IF NOT EXISTS spine_train
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

CREATE USER 'spine_user'@'localhost' IDENTIFIED BY '请替换为强密码';
GRANT ALL PRIVILEGES ON spine_train.* TO 'spine_user'@'localhost';
FLUSH PRIVILEGES;
```

或使用项目脚本仅建库（需 root 执行）：

```bash
mysql -u root -p < scripts/init-mysql.sql
```

### 4.3 连接串格式

写入 `server/.env` 的 `DATABASE_URL`：

```env
DATABASE_URL="mysql://spine_user:强密码@127.0.0.1:3306/spine_train"
```

## 五、MinIO 部署

### 5.1 方式 A：Docker（推荐）

在项目根目录已有 `docker-compose.yml`，生产机可只启动 **MinIO**（MySQL 若已独立安装可注释 compose 中的 mysql 服务）。

```bash
cd /opt/training-plan
# 仅启动 MinIO 与建桶任务（若 compose 含 mysql 且端口冲突，请单独编排 MinIO）
docker compose up -d minio createbuckets
```

默认：

- API：`http://服务器IP:9000`
- 控制台：`http://服务器IP:9001`（`minioadmin` / `minioadmin`，**上线后务必修改**）
- 桶名：`spine-train`（`createbuckets` 会自动创建并设为可下载）

生产请修改 `docker-compose.yml` 中 `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD`，并限制 9001 仅内网访问。

### 5.2 方式 B：二进制安装

参考 [MinIO 官方文档](https://min.io/docs/minio/linux/index.html) 安装为 systemd 服务，创建桶 `spine-train`，配置访问策略（v1 使用公网直链播放时需允许读对象）。

### 5.3 生产环境变量（`server/.env`）

```env
MINIO_ENDPOINT=127.0.0.1
MINIO_PORT=9000
MINIO_ACCESS_KEY=生产AccessKey
MINIO_SECRET_KEY=生产SecretKey
MINIO_BUCKET=spine-train
MINIO_USE_SSL=false
# 小程序播放视频用的 HTTPS 前缀（可与 Nginx 反代一致）
MINIO_PUBLIC_URL=https://cdn.example.com/spine-train
```

若通过 Nginx 将 `https://cdn.example.com` 反代到 MinIO 的 `/spine-train`，则 `MINIO_PUBLIC_URL` 填该前缀即可。

## 六、后端 API 部署

### 6.1 配置环境变量

```bash
cd /opt/training-plan/server
cp .env.example .env   # 若无 .env.example，可参考下文模板新建
nano .env
```

**生产 `.env` 模板（勿提交 Git）：**

```env
NODE_ENV=production
PORT=3000

DATABASE_URL="mysql://spine_user:强密码@127.0.0.1:3306/spine_train"

JWT_SECRET=请使用 openssl rand -base64 48 生成的随机串
JWT_EXPIRES_IN=7d

WECHAT_APPID=正式小程序AppID
WECHAT_SECRET=正式小程序AppSecret
WECHAT_SUBSCRIBE_TEMPLATE_ID=正式订阅消息模板ID

MINIO_ENDPOINT=127.0.0.1
MINIO_PORT=9000
MINIO_ACCESS_KEY=生产AccessKey
MINIO_SECRET_KEY=生产SecretKey
MINIO_BUCKET=spine-train
MINIO_USE_SSL=false
MINIO_PUBLIC_URL=https://cdn.example.com/spine-train
```

生成 JWT 密钥示例：

```bash
openssl rand -base64 48
```

### 6.2 安装依赖、生成 Prisma、执行迁移

```bash
cd /opt/training-plan/server
npm ci --omit=dev
# 若 npm ci 因无 lock 失败，使用：npm install --omit=dev
npm run prisma:generate
npx prisma migrate deploy
```

> `prisma` 在 devDependencies 中，若 `npm ci --omit=dev` 后无法执行 migrate，可临时安装：  
> `npm install prisma --no-save` 或在本机执行 migrate 后仅部署已迁移的库。

### 6.3 编译

```bash
npm run build
# 产物目录：server/dist/
```

### 6.4 启动（验证）

```bash
node dist/main.js
# 本机测试：curl http://127.0.0.1:3000/api/v1
```

API 全局前缀为 **`/api/v1`**（见 `server/src/main.ts`）。

### 6.5 使用 PM2 守护（推荐）

```bash
sudo npm install -g pm2

cd /opt/training-plan/server
pm2 start dist/main.js --name spine-train-api
pm2 save
pm2 startup   # 按提示执行命令，实现开机自启
```

常用命令：

```bash
pm2 status
pm2 logs spine-train-api
pm2 restart spine-train-api
```

### 6.6 服务器时区（订阅提醒依赖）

提醒任务按服务器本地时间的 `HH:mm` 匹配用户设置的 `remindTime`，请将时区设为上海：

```bash
sudo timedatectl set-timezone Asia/Shanghai
timedatectl
```

## 七、Nginx 与 HTTPS

### 7.1 安装 Nginx 与 Certbot

```bash
sudo apt-get install -y nginx certbot python3-certbot-nginx
```

### 7.2 站点配置示例

创建 `/etc/nginx/sites-available/spine-train-api`：

```nginx
server {
    listen 80;
    server_name api.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate     /etc/letsencrypt/live/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.example.com/privkey.pem;

    client_max_body_size 100m;

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

申请证书并启用站点：

```bash
sudo certbot --nginx -d api.example.com
sudo ln -s /etc/nginx/sites-available/spine-train-api /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### 7.3 MinIO 公网反代（可选）

若希望视频走 `https://cdn.example.com` 而非直接暴露 9000 端口，可增加 server 块将路径反代到 MinIO（需与 `MINIO_PUBLIC_URL` 一致），并在微信公众平台配置 **downloadFile** 合法域名为 `cdn.example.com`。

## 八、微信小程序配置

### 8.1 修改小程序配置后重新编译上传

| 文件 | 生产配置 |
|------|----------|
| `miniapp/src/manifest.json` → `mp-weixin.appid` | 正式小程序 AppID |
| `miniapp/src/config.js` → `API_BASE` | `https://api.example.com/api/v1` |
| `miniapp/src/config.js` → `SUBSCRIBE_TEMPLATE_ID` | 与 `server/.env` 中模板 ID 一致 |

本地编译发行包：

```bash
cd miniapp
npm install
npm run build:mp-weixin
```

用微信开发者工具打开 `miniapp/dist/build/mp-weixin`，上传代码并提交审核。

### 8.2 微信公众平台（正式）

登录 [微信公众平台](https://mp.weixin.qq.com/)：

1. **开发 → 开发管理 → 开发设置 → 服务器域名**
   - **request 合法域名**：`https://api.example.com`
   - **downloadFile 合法域名**：MinIO/CDN 域名（若视频直链非公网 API）
2. **功能 → 订阅消息**：使用与后端相同的模板 ID。

## 十一、安全建议

- `server/.env`、数据库密码、MinIO 密钥、微信 AppSecret **不得**进入 Git。
- `JWT_SECRET`、MySQL、MinIO 禁止使用默认值（如 `change-me-in-production`、`minioadmin`）。
- MySQL 仅监听 `127.0.0.1`；API 仅通过 Nginx 对外。
- 定期 `apt update && apt upgrade`，关注 Node / MySQL 安全公告。
- MinIO 控制台（9001）不要对公网开放，或加 IP 白名单 / VPN。

## 十二、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 小程序 request 失败 | 域名未备案、非 HTTPS、未配合法域名 | 检查 `API_BASE` 与公众平台域名 |
| 502 Bad Gateway | PM2 未启动或端口不对 | `pm2 logs`，确认 `PORT=3000` |
| 视频无法播放 | `MINIO_PUBLIC_URL` 错误或 downloadFile 未配置 | 统一 HTTPS 前缀并配置合法域名 |
| 登录失败 401 / 微信错误 | AppID/AppSecret 与小程序不一致 | 核对 `WECHAT_*` 与 manifest |
| 提醒不发送 | 未配置模板 ID、用户无订阅额度、时区不对 | 检查 `.env` 与 `timedatectl` |
| Prisma 迁移失败 | 数据库账号权限不足 | 确认对 `spine_train` 有 DDL 权限 |
| `npm run build` 报错缺少 `@prisma/client` | 未 generate | 先执行 `npm run prisma:generate` |
