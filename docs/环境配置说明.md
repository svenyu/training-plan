# 环境配置说明（开发 / 正式）

本文档汇总 **微信小程序 AppID/密钥**、**MySQL 数据库**、**MinIO 对象存储** 在本项目中的配置位置与示例值。  
**请勿将真实密钥提交到 Git**；`server/.env` 已在 `.gitignore` 中，仅保留 `server/.env.example` 作模板。

---

## 一、配置总览

| 配置项 | 开发环境主要位置 | 正式环境主要位置 | 说明 |
|--------|------------------|------------------|------|
| 小程序 AppID | `miniapp/src/manifest.json` | 同左（发布前改好） | 微信开发者工具识别的小程序 ID |
| 小程序密钥 AppSecret | `server/.env` | 服务器上的 `server/.env` 或进程环境变量 | **仅后端**，用于 `code2session` 登录 |
| 订阅消息模板 ID | `server/.env` + `miniapp/src/config.js` | 同左 | 两处需填**同一**模板 ID |
| 数据库连接 | `server/.env` → `DATABASE_URL` | 生产库连接串，写在部署机 `.env` | Prisma 读取 |
| MinIO | `server/.env` 一组 `MINIO_*` | 生产 MinIO 域名与密钥 | 训练视频存储 |

## 二、微信小程序 AppID 与密钥

### 2.1 从哪里获取

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. **开发** → **开发管理** → **开发设置**
3. 记录：
   - **AppID（小程序 ID）**
   - **AppSecret（小程序密钥）** — 仅后端使用，勿写入小程序前端代码

订阅消息模板：**功能** → **订阅消息** → 申请模板 → 复制 **模板 ID**。

### 2.2 开发环境

| 配置项 | 文件路径 | 键名 / 字段 | 示例 |
|--------|----------|-------------|------|
| 小程序 AppID（编译用） | `miniapp/src/manifest.json` | `mp-weixin.appid` | `wx1234567890abcdef` |
| 后端登录密钥 | `server/.env` | `WECHAT_APPID` | 与上面 AppID 相同 |
| 后端登录密钥 | `server/.env` | `WECHAT_SECRET` | 公众平台复制的 AppSecret |
| 订阅消息（后端发信） | `server/.env` | `WECHAT_SUBSCRIBE_TEMPLATE_ID` | 模板 ID，如 `xxxxxxxx` |
| 订阅消息（前端授权） | `miniapp/src/config.js` | `SUBSCRIBE_TEMPLATE_ID` | 与后端 **相同** |
| API 地址（小程序请求后端） | `miniapp/src/config.js` | `API_BASE` | `http://localhost:3000/api/v1` 或局域网 IP |

**代码引用位置：**

- 后端读取：`server/src/auth/wechat.service.ts`（`WECHAT_APPID`、`WECHAT_SECRET`）
- 后端发订阅消息：`server/src/remind/remind.service.ts`（`WECHAT_SUBSCRIBE_TEMPLATE_ID`）
- 小程序请求：`miniapp/src/utils/request.js` → `import { API_BASE } from '../config'`
- 小程序订阅授权：`miniapp/src/utils/subscribe.js` → `SUBSCRIBE_TEMPLATE_ID`

**模板文件：**

- `server/.env.example`（第 5–7 行）
- `miniapp/src/manifest.json`（第 8–9 行 `mp-weixin.appid`）

### 2.3 正式环境

| 配置项 | 文件 / 位置 | 说明 |
|--------|-------------|------|
| AppID | `miniapp/src/manifest.json` → `mp-weixin.appid` | 使用**正式小程序**的 AppID，重新编译并上传 |
| AppSecret | 生产服务器 `server/.env` 或容器环境变量 | `WECHAT_APPID`、`WECHAT_SECRET` |
| 订阅模板 | `server/.env` + `miniapp/src/config.js` | 正式环境申请的模板 ID |
| API 地址 | `miniapp/src/config.js` → `API_BASE` | 必须为 **HTTPS** 域名，如 `https://api.example.com/api/v1` |

**微信公众平台（正式）还需配置：**

- **开发** → **开发管理** → **服务器域名**：request 合法域名填 API 域名；若视频走 MinIO 公网地址，需配置 **downloadFile** 合法域名（或使用 CDN/反代到已备案域名）
- 正式版小程序不能使用开发者工具「不校验合法域名」

## 三、MySQL 数据库

### 3.1 库名与初始化

| 项目 | 值 / 位置 |
|------|-----------|
| 数据库名 | `spine_train` |
| 字符集 | `utf8mb4` / `utf8mb4_unicode_ci` |
| 建库 SQL 脚本 | `scripts/init-mysql.sql` |
| ORM 模型 | `server/prisma/schema.prisma`（`datasource db` 使用环境变量 `DATABASE_URL`） |
| 迁移命令 | 在 `server` 目录执行 `npx prisma migrate deploy` |

### 3.2 开发环境

| 配置项 | 文件路径 | 键名 | 示例 |
|--------|----------|------|------|
| 连接串 | `server/.env` | `DATABASE_URL` | `mysql://root:你的密码@localhost:3306/spine_train` |

**连接串格式：**

```text
mysql://<用户名>:<密码>@<主机>:<端口>/<数据库名>
```

**常见开发场景：**

| 场景 | `DATABASE_URL` 示例 |
|------|---------------------|
| 本机 MySQL，root 密码 `root` | `mysql://root:root@localhost:3306/spine_train` |
| Docker Compose 起 MySQL（见根目录 `docker-compose.yml`） | `mysql://root:root@localhost:3306/spine_train`（容器映射 3306，密码见 compose 中 `MYSQL_ROOT_PASSWORD`） |

**Docker MySQL 定义位置：** 根目录 `docker-compose.yml` → `services.mysql.environment`（`MYSQL_ROOT_PASSWORD`、`MYSQL_DATABASE`）

**本地安装说明：** `docs/本地安装-无Docker.md` 第一节

### 3.3 正式环境

| 配置项 | 位置 | 示例 |
|--------|------|------|
| 连接串 | 生产机 `server/.env` 或密钥管理服务 | `mysql://spine_user:强密码@db.internal:3306/spine_train` |

建议：

- 使用独立数据库账号，勿用 root
- 设置强密码；`JWT_SECRET` 同步改为随机长串（`server/.env`）
- 部署前执行：`cd server && npx prisma migrate deploy`

## 四、MinIO 对象存储

### 4.1 用途与默认桶

- 存储训练动作 **视频文件**
- 默认桶名：**`spine-train`**
- 后端读取逻辑：`server/src/upload/minio.service.ts`

### 4.2 环境变量（均在 `server/.env`）

| 变量名 | 含义 | 开发示例 | 正式示例 |
|--------|------|----------|----------|
| `MINIO_ENDPOINT` | MinIO 主机名（不含协议） | `localhost` | `minio.example.com` 或内网 IP |
| `MINIO_PORT` | API 端口 | `9000` | `9000` 或 `443`（若反代） |
| `MINIO_ACCESS_KEY` | 访问密钥 | `minioadmin` | 生产创建的 Access Key |
| `MINIO_SECRET_KEY` | 秘密密钥 | `minioadmin` | 生产 Secret Key |
| `MINIO_BUCKET` | 桶名 | `spine-train` | `spine-train` |
| `MINIO_USE_SSL` | 是否 HTTPS | `false` | `true`（公网建议） |
| `MINIO_PUBLIC_URL` | 小程序/浏览器播放视频的**公网前缀** | `http://localhost:9000/spine-train` | `https://cdn.example.com/spine-train` |

**模板位置：** `server/.env.example` 第 9–15 行

### 4.3 开发环境启动方式

**方式 A：脚本（无 Docker）**

| 项目 | 位置 / 值 |
|------|-----------|
| 启动脚本 | `scripts/start-minio.bat` |
| 可执行文件 | `scripts/minio.exe` 或系统 PATH 中的 `minio` |
| 数据目录 | `data/minio/`（脚本自动创建） |
| 控制台 | http://localhost:9001 |
| API | http://localhost:9000 |
| 默认账号密码 | `minioadmin` / `minioadmin`（脚本内 `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD`） |

首次需在控制台创建桶 **`spine-train`**，开发期可将桶策略设为 **public** 便于播视频。

**方式 B：Docker Compose**

| 项目 | 位置 |
|------|------|
| 编排文件 | 根目录 `docker-compose.yml` → `services.minio` |
| 根账号 | `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD` = `minioadmin` / `minioadmin` |
| 自动建桶 | `services.createbuckets` 会创建 `spine-train` 并设为可下载 |

### 4.4 正式环境

| 项目 | 建议 |
|------|------|
| 部署 | 独立 MinIO 集群或云对象存储；修改 `server/.env` 中全部 `MINIO_*` |
| `MINIO_PUBLIC_URL` | 使用 **HTTPS** 且小程序已配置 **downloadFile 合法域名** 的地址 |
| 桶权限 | 建议 **私有** + 后端签名 URL（当前 v1 为 public 前缀直链，上线前需评估） |
| 密钥 | 勿使用 `minioadmin`，在 MinIO 控制台创建专用用户 |

### 4.5 真机调试注意

手机无法访问电脑的 `localhost`。开发时需将：

- `miniapp/src/config.js` → `API_BASE` 改为电脑局域网 IP，如 `http://192.168.1.100:3000/api/v1`
- `server/.env` → `MINIO_PUBLIC_URL` 改为 `http://192.168.1.100:9000/spine-train`

并在微信开发者工具勾选「不校验合法域名」（仅开发）。

## 五、开发 vs 正式对照表

### 5.1 后端 `server/.env`

复制模板：`copy server\.env.example server\.env`（Windows）或 `cp server/.env.example server/.env`（Linux）

| 变量 | 开发环境 | 正式环境 |
|------|----------|----------|
| `NODE_ENV` | 可不设或 `development` | `production` |
| `PORT` | `3000` | `3000`（或由 Nginx 反代） |
| `DATABASE_URL` | `mysql://root:本地密码@localhost:3306/spine_train` | 生产 MySQL 连接串 |
| `JWT_SECRET` | `change-me-in-production` | **必须**改为随机强密钥 |
| `JWT_EXPIRES_IN` | `7d` | 按安全策略调整 |
| `WECHAT_APPID` | 开发/体验小程序 AppID | 正式小程序 AppID |
| `WECHAT_SECRET` | 对应 AppSecret | 正式 AppSecret |
| `WECHAT_SUBSCRIBE_TEMPLATE_ID` | 测试模板 ID | 正式模板 ID |
| `MINIO_ENDPOINT` | `localhost` | 生产 MinIO 域名 |
| `MINIO_PORT` | `9000` | `9000` 或反代端口 |
| `MINIO_ACCESS_KEY` | `minioadmin` | 生产密钥 |
| `MINIO_SECRET_KEY` | `minioadmin` | 生产密钥 |
| `MINIO_BUCKET` | `spine-train` | `spine-train` |
| `MINIO_USE_SSL` | `false` | `true`（推荐） |
| `MINIO_PUBLIC_URL` | `http://localhost:9000/spine-train` | `https://你的域名/spine-train` |

### 5.2 小程序端

| 文件 | 开发 | 正式 |
|------|------|------|
| `miniapp/src/manifest.json` → `mp-weixin.appid` | 测试 AppID | 正式 AppID |
| `miniapp/src/config.js` → `API_BASE` | `http://localhost:3000/api/v1` 或局域网 IP | `https://api.你的域名/api/v1` |
| `miniapp/src/config.js` → `SUBSCRIBE_TEMPLATE_ID` | 测试模板 ID | 正式模板 ID |

编译产物目录（微信开发者工具打开）：`miniapp/dist/dev/mp-weixin`（开发）/ `miniapp/dist/build/mp-weixin`（发行构建）

## 七、配置检查清单

**开发联调前：**

- [ ] `server/.env` 已填写 `WECHAT_APPID`、`WECHAT_SECRET`
- [ ] `miniapp/src/manifest.json` 中 `mp-weixin.appid` 与 AppID 一致
- [ ] MySQL 已建库 `spine_train`，`DATABASE_URL` 可连接
- [ ] MinIO 已启动，桶 `spine-train` 已创建
- [ ] `npm run start:dev` 后 API 可访问
- [ ] 真机调试时已改 `API_BASE`、`MINIO_PUBLIC_URL` 为局域网 IP

**正式上线前：**

- [ ] 使用正式 AppID / AppSecret / 订阅模板
- [ ] `JWT_SECRET`、数据库密码、MinIO 密钥已更换为强密码
- [ ] `API_BASE`、`MINIO_PUBLIC_URL` 为 HTTPS 且已在微信公众平台配置合法域名
- [ ] `npx prisma migrate deploy` 已在生产库执行
- [ ] 确认 `server/.env` 未进入版本库
