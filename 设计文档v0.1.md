这是一份“脊柱侧弯康复训练小程序”详细设计文档。这份设计兼顾了实用性、开发便捷性以及你要求的技术栈（Node.js + uni-app + MySQL）。

### 📋 一、 项目整体架构与技术选型

*   **前端（小程序端）**：uni-app（Vue3语法），一套代码可编译为微信小程序，开发效率高。
*   **后端（API服务）**：Node.js + Express（或轻量级的 Fastify），负责处理业务逻辑、数据接口和微信登录鉴权。
*   **数据库**：MySQL，配合可视化工具（如 Navicat）进行数据维护。
*   **媒体存储**：动作视频建议存放在对象存储（如阿里云OSS、腾讯云COS）或使用云存储，数据库中只保存视频的 URL 链接，以保证小程序加载速度。

### 🗃️ 二、 数据库设计 (MySQL)

为了满足你的需求，我们需要设计以下 4 张核心表：

**1. 动作库表 (`exercises`)**
用于维护基础的康复动作信息。
*   `id` (主键)
*   `name` (动作名称，如：猫式伸展)
*   `description` (动作说明/文字指导)
*   `video_url` (动作演示视频链接)
*   `category` (分类，如：热身、核心训练、拉伸)
*   `created_at` (创建时间)

**2. 训练计划表 (`training_plans`)**
用于定制具体的训练方案。
*   `id` (主键)
*   `plan_name` (计划名称，如：第一阶段21天矫正计划)
*   `start_date` (开始日期)
*   `end_date` (结束日期)
*   `total_days` (总天数，如：21天)
*   `status` (状态：进行中、已完成、未开始)
*   `user_id` (关联用户，方便未来给家人也创建计划)

**3. 计划动作关联表 (`plan_exercises`)**
用于维护某个计划中包含的动作、顺序及具体的组次要求。
*   `id` (主键)
*   `plan_id` (关联训练计划表)
*   `exercise_id` (关联动作库表)
*   `sort_order` (动作顺序，如：1, 2, 3)
*   `sets_and_reps` (组次说明，如：10次/组 * 3组)
*   `duration` (持续时间，如果有时间要求可填)

**4. 训练记录表 (`training_records`)**
用于记录每天的执行情况。
*   `id` (主键)
*   `plan_id` (关联训练计划)
*   `user_id` (关联用户)
*   `record_date` (训练日期，如：2026-05-18)
*   `rating` (本次评价：优、良、中、差)
*   `remark` (备注，比如今天腰部有点酸)
*   `created_at` (提交时间)

### ⚙️ 三、 后端接口设计 (Node.js + Express)

后端主要提供 RESTful API 供前端调用：

1.  **微信授权登录模块**
    *   `POST /api/auth/login`：接收前端传来的微信 `code`。后端调用微信官方接口 (`code2Session`) 获取用户的 `openid`。如果是第一次登录，自动在用户表中创建记录，并返回自定义的登录态（如 JWT Token）给前端。
2.  **动作管理模块**
    *   `GET /api/exercises`：获取所有康复动作列表（供后台维护或前端查看）。
    *   `POST /api/exercises`：新增/更新动作（包含视频上传后的URL保存）。
3.  **训练计划模块**
    *   `GET /api/plans/current`：获取当前正在进行的训练计划详情（包含计划基本信息以及关联的动作列表，按 `sort_order` 排序）。
    *   `POST /api/plans`：创建或更新定制计划（选择动作、设置组次、起止时间）。
4.  **训练执行与记录模块**
    *   `POST /api/records`：提交当天的训练记录（包含评分和备注）。
    *   `GET /api/records/history`：获取历史训练记录，用于查看坚持的天数和评价趋势。

### 📱 四、 前端页面与交互设计 (uni-app)

小程序端建议分为三个核心页面（TabBar）：

**1. 首页 / 训练执行页 (核心功能)**
*   **展示**：显示当前正在进行的计划名称和剩余天数（如：21天计划 - 第5天）。
*   **动作列表**：按顺序展示今日需要完成的动作卡片。每个卡片包含动作名称、组次要求。
*   **执行交互**：
    *   点击某个动作，进入**动作详情页**。页面中间播放 `video_url` 对应的视频，下方展示 `description` 动作说明。
    *   页面底部有一个大大的“完成本组/完成动作”按钮。
*   **打卡提交**：当所有动作完成后，弹出一个打分面板（优、良、中、差）和备注输入框，点击“提交”后调用后端接口保存记录，并更新首页的完成状态。

**2. 计划定制页 (家长端/管理端)**
*   **动作选择**：从动作库中勾选需要加入本次计划的康复动作。
*   **参数设置**：为每个选中的动作设置 `sort_order`（拖拽排序或输入数字）和 `sets_and_reps`（如输入“15次*3组”）。
*   **周期设置**：选择开始和结束日期，生成计划。

**3. 我的 / 记录页**
*   **日历视图**：用日历展示当月的训练打卡情况（打勾表示已完成）。
*   **历史列表**：展示过往的训练记录，点击可查看当天的具体评价和备注。
*   **登录**：页面加载时静默调用 `uni.login` 获取 code 并换取登录态。

### 🔐 五、 微信授权登录具体实现思路

1.  **前端 (uni-app)**：
    在小程序启动（`App.vue` 的 `onLaunch`）或进入“我的”页面时，调用 `uni.login()` 获取临时的登录凭证 `code`。将这个 `code` 通过 `uni.request` 发送给你的 Node.js 后端。
2.  **后端 (Node.js)**：
    拿到 `code` 后，配合你的小程序 `AppID` 和 `AppSecret`，向微信服务器发起请求换取用户的唯一标识 `openid`。
    *   如果该 `openid` 在数据库中已存在，直接生成 Token 返回。
    *   如果不存在，视为新用户，在数据库创建一条新用户记录，再生成 Token 返回。
3.  **前端存储**：
    拿到后端返回的 Token 后，使用 `uni.setStorageSync` 保存在本地。之后每次请求业务接口（如提交训练记录）时，在 HTTP Header 中带上这个 Token，后端以此识别是“你闺女”的训练数据。

### 💡 六、 进阶建议（让小程序更好用）

1.  **视频自动播放**：在动作详情页，可以设置视频进入页面后自动静音播放，减少操作步骤。
2.  **语音/文字提醒**：利用微信小程序的“订阅消息”功能，每天晚上固定时间推送一条消息：“宝贝，今天的脊柱侧弯训练还没有完成哦，加油！”。
3.  **数据可视化**：在记录页，可以用简单的柱状图展示一周的训练评分变化，给孩子正向的反馈和激励。
